# /fc — FCstack

Startup wisdom, deployed at 2am.

**[Live Site](https://vedsoni-dev.github.io/FCstack/)**

5 Claude Code skills that read your codebase and give you founder-level advice. Mock interviews. Application builder. Office hours. Demo Day coach. All context-aware. All in your terminal.

## Install

Paste this into Claude Code:

```
Install FCstack skills from https://github.com/VedSoni-dev/FCstack — clone the repo, then copy each skill folder from skills/ into ~/.claude/skills/
```

Then run `/fc` in any project.

## Skills

| Command | What it does |
|---|---|
| `/fc` | Full startup review — scores idea, market, product, moat, growth, team. Gives a verdict. |
| `/fc review` | Deep analysis with specific feedback from your codebase |
| `/fc roast` | Brutal, honest feedback. The thing you know but won't admit. |
| `/fc pitch` | Pitch review through investor lens |
| `/fc metrics` | Find growth signals in your code |
| `/fc-apply` | Interactive application builder. 17 questions, one at a time. Drafts from your code. |
| `/fc-interview` | Mock 10-minute interview. Two partners grill you. Scorecard + verdict. |
| `/fc-office-hours` | Pick a mentor. Bring a problem. Get homework. |
| `/fc-demo-day` | 60-second pitch from your codebase. Scores each section. |

## How it works

1. You run `/fc` in any project directory
2. It silently reads your README, package.json, git history, routes, models
3. It gives you specific, scored, actionable feedback based on YOUR code
4. Every answer cites real startup essays and frameworks
5. Every session ends with "do this TODAY"

## The mentors

Skills simulate real startup mentors with distinct personalities:

- **Michael Seibel** — Users & simplicity. "Explain it like I'm not technical."
- **Dalton Caldwell** — Moats & business model. "Waitlists don't count. Ship it."
- **Gustaf Alstromer** — Growth & metrics. "What's your activation rate? Not signups."
- **Jared Friedman** — Technical architecture. "Show me the schema. Not the pitch deck."
